# 갈림길 장면 일러스트 교체 가이드

설문 화면(`/survey`)의 각 문항은 3개 레이어로 구성된 1인칭 갈림길 장면입니다.
이 폴더에 아래 네이밍 규칙대로 이미지를 넣으면 **코드 수정 없이** SVG 플레이스홀더가
자동으로 실제 일러스트로 교체됩니다. (이미지가 없거나 로드 실패 시 플레이스홀더 유지)

## 파일 네이밍 규칙

| 파일명 | 레이어 | 역할 | 패럴랙스 |
|---|---|---|---|
| `background-invest-clean.png` | 배경(공통) | 전 문항 공통 배경 — **길이 그려지지 않은** 하늘/원경/들판 아트 | 움직임 가장 적음 |
| `q1-background.png` | 배경(문항별) | 특정 문항만 다른 배경을 쓸 때 (공통보다 우선) | 움직임 가장 적음 |
| `q1-road.png` | 길 | 좌우로 갈라지는 두 갈래 길 | 선택 시 확대 중심 |
| `q1-foreground.png` | 전경 | 바닥/근경 (수풀, 잔돌) | 움직임 가장 큼 |

- **공통 배경(`background-invest-clean.png`)**: Y자 갈림길이 **이미지 자체에 포함**된
  버전을 넣습니다(코드로 길을 그리지 않음). `object-fit: cover` + `object-position:
  center bottom`으로 배치되어, 화면 비율이 바뀌어도 이미지 하단(길·초원)이 항상 보이고
  분기점이 잘리지 않습니다(세로 화면에서는 좌우가 잘림).
- **캐릭터-길 정렬**: 캐릭터의 시작점·분기점·갈래길 끝은 `app/survey/ForkScene.tsx`의
  `ROAD` 상수에 **이미지 기준 비율(0~1)**로 정의됩니다. 배경 이미지의 길 위치가
  다르면 `ROAD`의 `start`/`fork`/`leftEnd`/`rightEnd` 값(fx,fy)만 조정하세요.
  이미지 렌더 영역을 화면 크기에서 실시간 계산하므로, 창 크기가 바뀌어도 캐릭터가
  항상 이미지 속 같은 길 지점에 위치합니다.

- `q1` 자리에 문항 ID(`q1` ~ `q12`)를 넣습니다. 문항 ID는 `data/questions.json`의 `id`와 동일합니다.
- 경로 규칙은 `lib/scenes.ts`의 `getSceneAssets()`에 정의되어 있습니다.
- 각 레이어는 화면 비율에 맞춰 늘려서(stretch) 표시되므로, 구도가 중요한 요소는
  화면 기준 상대 위치로 그려주세요. 현재 구도: 지평선 ≈ 세로 47%, 갈림길 분기점 ≈
  세로 58%, 캐릭터(코드로 렌더링됨)는 트렁크 길 아래쪽(50%, 90%)에서 시작해
  분기점까지 직진한 뒤 선택한 갈래길로 걸어 들어갑니다.

## 이미지 권장 스펙

- **크기**: 세로형 1200×1800px 이상 (모바일 풀스크린 기준, `object-fit: cover`로 채워짐)
- **포맷**: PNG 또는 WebP
- **투명 배경**: `road`, `foreground` 레이어는 반드시 투명 배경으로 제작
  (뒤 레이어가 비쳐야 깊이감이 생깁니다. `background`는 불투명이어도 됨)
- **여백(bleed)**: 각 레이어는 화면보다 12% 크게 렌더링된 뒤 확대/이동 애니메이션이
  적용되므로, 가장자리가 잘려도 어색하지 않게 여유를 두고 그려주세요.
- 선택지 버튼이 화면 세로 40% 지점 좌우에 오버레이되므로, 그 위치에 두 갈래 길이
  오도록 구도를 잡으면 가장 자연스럽습니다.
