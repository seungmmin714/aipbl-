투자 성향 진단 & 자산 배분 추천 서비스 PRD (v2.1)

부록 A 상세 일정표 전면 개정본

작성 목적: 기존 PRD v2.0(투자 MBTI 모델 도입)을 기반으로 부록 A를 시간대별 × 직군별 × 완료조건 × 협업 핸드오프 형식으로 확장하고, 문서 전반의 유형·문항 수 표기를 투자 MBTI 모델(4축·16유형)에 맞춰 통일한 v2.1 문서


# 목차

목차는 Word에서 열면 자동 업데이트할 수 있습니다.

# 0. 개요

- 서비스 정의: 사용자가 10개 문항에 답하면 투자 성향 유형을 진단하고, 유형별 자산 배분 예시(포트폴리오)를 시각적으로 보여주는 웹 서비스.

- 포지셔닝: 실제 투자 자문/권유가 아닌, 교육·재미 목적의 성향 진단 시뮬레이션.

# 1. 문제 (누구의 불편인가)

- 투자를 처음 시작하려는 2030 사회초년생은 "내 성향에 맞는 자산 배분이 뭔지" 감이 없다.

- 기존 증권사 성향 진단은 딱딱하고 결과가 텍스트뿐이라 이해·공유가 어렵다.

- 감정: 막막함, 시작 자체를 미루게 됨.

# 2. 목표 사용자

- 1차 타깃: 투자 입문 단계의 2030 사회초년생.

- 사용 맥락: 재미로 성향 테스트 → 결과를 SNS/지인에게 공유.

- 노리는 행동: 결과 이미지 저장·공유를 통한 자연스러운 바이럴.

# 3. 핵심 기능 3개

- 기능1. 성향 진단: 10개 문항 응답 → 4개 축 점수 계산 → 16가지 투자 MBTI 유형 중 하나로 판정.

- 기능2. 자산 배분 예시: 판정된 유형에 맞는 정적 포트폴리오를 레이더/파이 차트로 시각화.

- 기능3. 결과 저장·공유: 결과 화면을 이미지로 저장(html2canvas) + 텍스트 복사 + Web Share API.

# 3-1. 투자 MBTI 진단 모델 (핵심 컨셉)

기존 단순 점수 합산(안정형/중립형/공격형) 방식을 확장하여, 사용자가 친숙한 MBTI 형식을 투자 성향에 적용한다. 4개의 투자 성향 축에서 각각 두 성향 중 하나로 판정하여, 총 16개의 ‘투자 MBTI 유형’으로 결과를 제시한다. 이는 재미와 공유 동기를 극대화하기 위한 핵심 차별화 요소다.

## 3-1-1. 4가지 투자 성향 축

| 축 | 코드 | 왼쪽 성향 | 오른쪽 성향 | 측정 내용 |
| --- | --- | --- | --- | --- |
| 위험 태도 | R/S | R (Risk-taking, 위험 감수) | S (Stable, 안정 지향) | 손실·변동성에 대한 태도 |
| 정보 처리 | D/I | D (Data, 데이터 기반) | I (Intuition, 직관 기반) | 투자 결정 시 근거의 유형 |
| 투자 기간 | L/T | L (Long-term, 장기) | T (Trading, 단기) | 목표 보유 기간·매매 빈도 |
| 자산 성향 | G/V | G (Growth, 성장주) | V (Value/Safe, 가치·안전자산) | 선호하는 자산군 스타일 |

## 3-1-2. 진단 로직 (축별 판정 방식)

10개 문항을 4개 축에 배분(축당 2~3문항)한다. 각 문항의 응답 점수를 해당 축에 누적하고, 축별 합산 점수가 기준선(중앙값)보다 크면 한쪽 성향, 작거나 같으면 반대 성향으로 판정한다. 4개 축의 알파벳을 조합해 최종 4글자 코드(예: R-D-L-G)를 생성하고, 이를 16개 유형 정의(mbti-types.json)에서 조회하여 결과와 추천 포트폴리오를 매핑한다.

## 3-1-3. 16개 투자 MBTI 유형 예시

| 코드 | 유형 별명 | 한 줄 설명 | 대표 추천 포트폴리오 방향 |
| --- | --- | --- | --- |
| RDLG | 공격형 전략가 | 데이터를 믿고 장기 성장주에 과감히 베팅 | 성장주·해외주식 비중 높음 |
| RILT | 감각형 트레이더 | 직관과 순발력으로 단기 매매를 즐김 | 테마주·단기 트레이딩 비중 |
| SDLV | 신중한 자산가 | 데이터 근거로 장기 가치·안전자산 축적 | 채권·배당주·가치주 중심 |
| SILV | 안심 지킴이 | 변동성을 피하고 원금 보존을 최우선 | 예금·채권 비중 최대 |
| (그 외 12개 유형은 4축 조합으로 자동 생성됨 — 문서에는 대표 4개만 예시로 제시) | - | - | - |

※ 위 4개는 대표 예시이며, 실제 서비스에서는 4축(R/S · D/I · L/T · G/V)의 조합으로 만들어지는 16개 유형 전체를 mbti-types.json에 정의한다.

## 3-1-4. 문항–축 매핑 예시

| 문항 번호 | 측정 축 | 예시 질문 |
| --- | --- | --- |
| Q1~Q3 | 위험 태도(R/S) | "투자 원금이 20% 손실 나면?" |
| Q4~Q5 | 정보 처리(D/I) | "종목을 고를 때 무엇을 먼저 보나요?" |
| Q6~Q8 | 투자 기간(L/T) | "얼마나 자주 계좌를 확인하나요?" |
| Q9~Q10 | 자산 성향(G/V) | "다음 중 더 끌리는 투자처는?" |

# 4. 안 할 것

- 로그인/회원가입, 백엔드 DB, 실시간 시세 데이터, AI/LLM API, 유료화, 외부 SNS SDK, 음성 기능.

# 5. 완료 기준

- (A) 기능: 10문항 진단→4축 판정→16개 투자 MBTI 유형 도출→차트 표시→이미지 저장/공유가 정상 동작.

- (B) 안정성: 모바일·PC 반응형, 주요 브라우저 정상 렌더링.

- (C) 품질·배포: Vercel 공개 URL 배포, 지인 10명 테스트 완료.

- (D) 측정: 결과 저장·공유 버튼 클릭 횟수를 localStorage 카운터로 최소 기록(외부 SDK 없이 바이럴 확산 정도 자체 확인).

## 6.4 데이터 구조 설계 (정적 JSON)

모든 데이터는 /data 폴더의 정적 JSON으로 관리하며, 3개 모델로 구성됩니다.

```
// data/questions.json  — 진단 문항
[
  {
    "id": "q1",
    "text": "투자 원금이 20% 손실 나면?",
    "options": [
      { "label": "즉시 매도한다",   "score": 1 },
      { "label": "일부 매도한다",   "score": 2 },
      { "label": "기다린다",        "score": 3 },
      { "label": "추가 매수한다",   "score": 4 }
    ]
  }
]
```


```
// data/types.json — 성향 유형(점수 구간별)
[
  { "type": "안정형",   "min": 0,  "max": 20, "desc": "..." },
  { "type": "중립형",   "min": 21, "max": 35, "desc": "..." },
  { "type": "공격형",   "min": 36, "max": 50, "desc": "..." }
]
```


```
// data/portfolios.json — 유형별 추천 자산배분
[
  { "type": "안정형", "allocation": { "예금": 60, "채권": 30, "주식": 10 } }
]
```

[투자 MBTI 확장] MBTI 방식 도입에 따라 아래 두 데이터 모델을 추가한다. 기존 types.json(점수 구간형)은 하위 호환을 위해 유지하되, MBTI 모드에서는 mbti-types.json을 사용한다.

```
// data/axes.json — 4개 투자 성향 축 정의
[
  { "axis": "risk",  "pos": "R", "neg": "S", "questions": ["q1","q2","q3"], "threshold": 8 },
  { "axis": "info",  "pos": "D", "neg": "I", "questions": ["q4","q5"],       "threshold": 5 },
  { "axis": "term",  "pos": "L", "neg": "T", "questions": ["q6","q7","q8"], "threshold": 8 },
  { "axis": "asset", "pos": "G", "neg": "V", "questions": ["q9","q10"],      "threshold": 5 }
]
```


```
// data/mbti-types.json — 16개 유형 정의
[
  {
    "code": "RDLG",
    "nickname": "공격형 전략가",
    "desc": "데이터 기반으로 장기 성장주에 과감히 투자하는 유형",
    "allocation": { "국내주식": 30, "해외주식": 40, "채권": 10, "현금": 20 }
  }
  // ... 나머지 15개 유형(4축 조합)
]
```

```
     │
     ▼
```

portfolios.json 조회 → 추천 자산배분

```
     │
     ▼
```

[Recharts] 레이더/파이 차트 렌더링

```
     │
```

## 6.8 투자 MBTI 진단 전용 기술 요소

MBTI 방식 도입으로 추가되는 기술·구현 요소를 정리한다. 신규 외부 의존성은 없으며, 기존 스택(Next.js·React·TypeScript·Recharts) 내에서 로직·데이터·컴포넌트만 확장한다.

## 6.8-1 추가/변경 기술 요소

| 구분 | 요소 | 내용 | 선정 이유 / 비고 |
| --- | --- | --- | --- |
| 진단 로직 | lib/mbti.ts (신규) | 축별 점수 누적·threshold 비교·4글자 코드 조합 함수 | 기존 score.ts를 대체가 아닌 확장. 순수 함수로 단위테스트 용이 |

## 6.8-2 API 관점 요약

투자 MBTI 도입에도 불구하고 외부 유료 API·서버 API·AI/LLM API는 여전히 사용하지 않는다. 판정은 rule-based 로직(lib/mbti.ts)으로 클라이언트에서 100% 처리되며, 사용하는 API는 기존 6.2와 동일(Web Share / Clipboard / Canvas / File·Blob)하다. 다만 16유형별 결과 공유를 위해 Next.js 동적 라우트(/result/[code])를 활용하는 점만 추가된다.
