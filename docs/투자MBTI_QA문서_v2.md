## 1. 재현 절차 (Critical / High)

### DEF-01. `다음` 버튼 연타

**전제**: `handleNext`의 유일한 가드는 `if (!selectedValue) return`. `currentIdx` 증가가 `setTimeout(250ms)`로 지연되는 동안 버튼이 계속 활성 상태다.

**재현 A — 크래시**
1. `/survey` 진입
2. Q1~Q10을 순서대로 답변
3. **Q11(인덱스 10)** 선택지 클릭
4. `다음` 버튼을 **250ms 이내에 2회** 클릭 (트랙패드 더블탭이면 충분)
5. **기대**: Q12로 1회만 이동
6. **실제**: `currentIdx`가 12가 되어 `questionsData[12] === undefined` → `q.text` 접근 시 `TypeError` → **흰 화면**

**재현 B — 조용한 오답 (더 위험)**
1. Q1~Q3에 `R, R, S` 응답 (위험감수 2:1)
2. Q4에 `D`, **Q5에서 `다음` 연타** → Q6으로 스킵
3. Q6에 `I` 응답
4. Q7~Q12 정상 응답
5. **기대**: 코드에 `D` 포함
6. **실제**: D 카운트가 2→1로 떨어져 축이 `I`로 반전. **에러도, 경고도 없음**

**영향도 전수조사**
```
4096개 응답 패턴 × 12개 스킵 위치 = 49,152건
→ 12,288건 (25.0%)에서 최종 코드가 달라짐
```
> 3:0으로 압도한 축은 문항 하나가 빠져도 `>= 2` 조건을 만족하므로 결과가 같다.
> **접전(2:1) 축에서 우세한 쪽이 빠질 때만** 뒤집힌다 → 버그가 75% 확률로 숨는다.

### DEF-03. 채점 함수 입력 검증 부재

`lib/mbti.ts`의 `calculateMBTI`에 아래를 넣으면 모두 **에러 없이 유효한 코드를 반환**한다.

| 입력 | 반환 | 정상 동작이라면 |
|---|---|---|
| `[]` | `"SITV"` | 예외 |
| 답변 1개 `[{q1:"R"}]` | `"SITV"` | 예외 |
| `"R"` 6개 중복 | `"RITV"` | 예외 (문항 중복) |
| `["X","Y","Z"] × 4` | `"SITV"` | 예외 (알 수 없는 값) |
| 한 축에 4문항, 다른 축에 2문항 | 코드 반환 | 예외 (축별 문항 수) |

원인: `const axis1 = R >= 2 ? "R" : "S"` — **카운트가 부족하면 무조건 반대편(보수 성향)을 고른다.** 모든 결손·오염 데이터가 최보수 유형 `SITV`로 수렴한다.

### DEF-04. 결과 코드 검증 미적용

1. 랜딩 하단 조회창에 `ㅁㄴㅇㄹ` 입력 → `조회`
2. **기대**: 입력창 아래 인라인 에러
3. **실제**: `/result/ㅁㄴㅇㄹ` 로 라우팅. 결과 페이지에서야 "찾을 수 없습니다"
4. 추가: `maxLength` 없음 → 5,000자 입력 시 초장문 URL 생성

> `isValidMBTICode()`가 `lib/mbti.ts`에 구현되어 있으나 **어디서도 호출되지 않는다.**

### DEF-06. 상태 휘발

| # | 절차 | 기대 | 실제 |
|---|---|---|---|
| 1 | Q7까지 답변 후 F5 | 답변 유지 또는 명시적 초기화 안내 | 조용히 Q1으로, 답변 전부 소실 |
| 2 | Q7에서 브라우저 뒤로가기 | Q6으로 이동 | **랜딩으로 이탈**, 전부 소실 |

원인: `useState`만 사용. 문항 이동이 라우팅이 아니라 상태 변경이라 히스토리 엔트리가 없다.

### DEF-09. 선택 하이라이트 잔상

1. Q4(`D`/`I` 축)에서 **첫 번째 선택지(`D`)** 클릭
2. `다음` 클릭
3. Q5가 나타나는 첫 프레임을 관찰 (개발자도구 → Rendering → Paint flashing, 또는 6× CPU 스로틀링)
4. **실제**: Q5의 `D` 선택지가 **선택된 상태로 파랗게 한 프레임 표시**된 뒤 해제

원인: `selectedValue`가 state이고 `useEffect`(페인트 이후 실행)로 동기화된다. 같은 축의 문항은 선택지 `value`를 공유(`D`/`I`)하므로 이전 값이 새 문항의 옵션과 일치해버린다.

---

### DEF-10. 선택지 순서 편향 (설계 결함)

| 축 | 첫 번째 선택지 배치 | 판정 |
|---|---|---|
| R/S | `[R, S, R]` | 혼합 ✅ |
| **D/I** | **`[D, D, D]`** | **3문항 전부 D 선점 ❌** |
| L/T | `[T, L, T]` | 혼합 ✅ |
| G/V | `[G, V, V]` | 혼합 ✅ |

초두 효과로 응답자는 첫 항목을 고르는 경향이 있다. D/I축만 편향이 걸려 **데이터기반(D)이 체계적으로 과대표집**된다.
